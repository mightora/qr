📦 Multi-Part QR

▼

For large data that won't fit in a single QR code, split it across multiple QR codes in a numbered sequence.

How it works

Each QR code is labelled with its position (e.g. "1/3") and includes a checksum so the receiver can verify all parts arrived correctly.

Smaller chunks = simpler QR codes = easier to scan. Larger chunks = fewer QR codes.

📚 Learn About QR Codes

Everything you need to know to create and use QR codes effectively

Technical: A QR (Quick Response) code is a 2D matrix barcode that stores data as a pattern of black and white squares arranged in a grid.

Plain English: It's like a barcode that your phone camera can read. You can pack a website address, contact details, WiFi credentials, or any text into one small image.

Technical: A QR code can hold up to 4,296 alphanumeric characters or 2,953 bytes of binary data, depending on the error correction level used.

Plain English: About 4,000 letters at most — roughly a short page of text. For bigger data, use the Multi-Part QR mode in this tool.

Tip: Shorter data = simpler, cleaner QR code = easier to scan from any distance.

Technical: QR codes use Reed-Solomon error correction, which allows damaged or obscured areas to be reconstructed. Levels are: L (7%), M (15%), Q (25%), H (30%).

Plain English: Even if part of the QR code is scratched, dirty, or covered — it can still be read. But higher protection means less room for data, making the QR denser.

Best practice: Use M (Medium) for most cases. Use H (High) if you plan to add a logo in the centre of the QR code.

  • Too much data — QR pattern becomes very dense
  • Low contrast between foreground and background colours
  • Logo or image covers too much of the QR pattern
  • Printed too small (minimum 2cm × 2cm recommended)
  • Damaged, dirty, or reflective surface
  • Unusual colour combinations that confuse scanner algorithms

Tip: Always test your QR code with multiple devices before using it in production.

  • Keep data as short as possible — use URL shorteners for long URLs
  • Use high contrast — dark foreground on light background is most reliable
  • Print at minimum 2cm × 2cm
  • Always include a quiet zone (white border) around the QR code
  • Test on multiple devices before publishing
  • Use error correction H if adding a logo overlay
  • Avoid inverting (light on dark) if possible — not all scanners support it

Technical: Large payloads are split into chunks, each encoded into a separate QR code with sequence metadata for ordered reconstruction.

Plain English: If your data is too big for one QR code, we create a numbered set of QR codes. The receiver scans them all in order and the data can be reassembled.

Format: Each QR begins with [QR:1/3:a1b2] — indicating position (1 of 3) and a checksum (a1b2) to verify completeness.

  • SVG — Vector format, scales to any size without quality loss. Best for web and design tools.
  • PNG — Raster image, ideal for email, documents and social media. Export at 2× or 4× for high resolution.
  • PDF — Print-ready document with the QR code centred and optionally labelled.
  • ZIP — Downloads all formats (PNG, PNG@2x, SVG) in a single archive.

Technical: Text files are UTF-8 encoded; binary files are base64 encoded. The content is wrapped in a JSON metadata envelope with filename, encoding type, and data fields.

Plain English: You can encode a small text file, JSON config, or CSV directly into a QR code. When someone scans it, they can extract and save the original file.

Limitations: QR codes have a ~2.9KB capacity. Files larger than this will need Multi-Part QR mode.
